تفاوت Git و GitHub چیست؟
Gitیک سیستم کنترل نسخه (Version Control System) است که روی کامپیوتر خودت نصب می‌شود و برای مدیریت تغییرات کد استفاده می‌شود.
GitHub یک سرویس آنلاین است که مخازن Git را روی اینترنت میزبانی می‌کند و امکان همکاری تیمی، Pull Request، Issue و… را می‌دهد.
به‌طور ساده: Git ابزار است، GitHub سرویس آنلاین.
Merge و Rebase چه تفاوتی دارند؟
Merge تاریخچه‌ی شاخه‌ها را حفظ می‌کند و یک commit جدید برای ادغام می‌سازد.
Rebase تاریخچه را خطی و تمیز می‌کند و commitها را روی شاخه‌ی مقصد بازنویسی می‌کند.
Merg برای کار تیمی امن تر است.
 Rebase تمیزتر ولی خطرناک‌تر اگر روی branch مشترک انجام شود
Conflict چگونه به وجود می‌آید و چگونه حل می‌شود؟
Conflict زمانی ایجاد می‌شود که:
دو نفر یک بخش یکسان از یک فایل را تغییر داده باشند
Git نتواند تشخیص دهد کدام تغییر درست است
روش حل:
فایل conflictدار را باز می‌کنیم
کدهای علامت‌گذاری‌شده (<<<<, ====, >>>>) را بررسی می‌کنیم
نسخه درست را انتخاب یا ترکیب می‌کنیم
فایل را ذخیره و:
git add file
git commit
چرا نباید مستقیماً روی branch اصلی (main) کار کنیم؟
احتمال خراب شدن پروژه بالا می‌رود
مدیریت تغییرات سخت می‌شود
در پروژه‌های تیمی باعث تداخل می‌شود
بهترین روش:
ساخت branch جدا
توسعه روی آن
سپس merge به main
دستور git-fetch چه تفاوتی با git-pull دارد؟
git fetch فقط اطلاعات جدید را می‌گیرد ولی کد را تغییر نمی‌دهد
git pull = fetch + merge
 fetch امن‌تر است چون تغییرات را قبل از ادغام می‌بینی
چرا نوشتن commit message خوب مهم است؟ (با مثال)
فهم تاریخچه پروژه
راحت‌تر دیباگ کردن
همکاری بهتر در تیم
